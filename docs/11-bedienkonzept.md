# Bedienkonzept

## Grundsatz

Der Einstieg muss einfach bleiben. Erweiterte Faktoren sind optional und dürfen die erste Nutzung nicht blockieren.

## Minimaler Ablauf

- Insel anlegen
- DLCs auswählen
- Cluster erstellen
- Gebäude auswählen
- Anzahl und Effizienz eingeben
- Ergebnis ansehen
- Prognose erstellen

## Hauptbereiche

- Start und Inselauswahl
- Ist-Stand
- Cluster
- Produktionsketten
- Prognosen
- Vergleich
- Diagnose
- Beobachtungen
- Einstellungen
- Import und Export

## Gebäudeeingabe

Mehrere gleiche Gebäude können zusammengefasst werden.

Einzelne Erfassung ist möglich, wenn Effizienz, Beschäftigung oder Entfernung abweichen.

## Progressive Offenlegung

Standardmäßig sichtbar:

- Gebäude
- Anzahl
- Cluster
- Effizienz
- Beschäftigung

Optional aufklappbar:

- Arbeitsmodus
- Upgrades
- Warenpriorität
- Lagerwerte
- Entfernungen
- Beobachtungsdaten

## Ergebnisdarstellung

Die Anwendung soll zuerst verständliche Aussagen zeigen und danach Detailwerte.

Beispiel:

- Rumbrennerei nur zu 72 Prozent versorgt
- eine zusätzliche Zuckerplantage empfohlen
- Transportlage unklar, da keine Entfernung oder Lagerbeobachtung vorliegt

## Visualisierung

Sinnvoll:

- Tabellen
- Kennzahlen
- Differenzanzeigen
- Statuskennzeichen
- einfache Flussdarstellung

Nicht für die erste Version:

- grafischer Karteneditor
- vollständig dynamische Netzwerkanordnung
